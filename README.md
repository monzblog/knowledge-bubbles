# Knowledge Bubbles — 知識の宇宙

知識トピックを宇宙のバブルとして可視化するインタラクティブWebアプリ。  
`topics.js` のデータを AI で大量生成し、継続的に拡張できます。

---

## ファイル構成

```
knowledge-bubbles/
├── index.html               # メインアプリ（Canvas + ラベルレイヤー）
├── topics.js                # 知識データベース（window.TOPICS_DB）
└── generate-topics-prompt.md  # AI生成用プロンプト
```

---

## トピックデータの形式

`topics.js` の各エントリは 4 フィールドの配列です。

```js
["キーワード", "カテゴリ", 重み, "タグ1 タグ2 タグ3"]
```

| フィールド | 型 | 説明 |
|---|---|---|
| キーワード | string | 日本語の概念・用語（カタカナ・英字混在OK） |
| カテゴリ | string | 主カテゴリ（固定リストから選択） |
| 重み | number 1–10 | バブルのサイズ。10=最重要、1=超ニッチ |
| タグ | string | スペース区切り2〜4個（固定リストから選択） |

---

## プロンプトの作り方

`generate-topics-prompt.md` を AI に渡すことで、3000 件規模のデータを自動生成できます。  
以下に、効果的なプロンプト設計のポイントをまとめます。

---

### 1. 出力フォーマットを完全に固定する

AIが迷わないよう、フォーマットを例示付きで明示します。

```
フォーマット（1エントリあたり）:
["キーワード","カテゴリ",重み,"タグ1 タグ2 タグ3"]

例:
["量子もつれ","物理",8,"物理 情報 哲学"]
["プロンプトエンジニアリング","AI",6,"AI 言語 テック"]
```

> **ポイント:** 選択肢（カテゴリ一覧・タグ一覧）を明記し、「必ず以下から選ぶ」と制約します。  
> 自由記述を許すと表記ゆれが発生し、フィルタリングが壊れます。

---

### 2. 深さの分布を数値で指定する

「専門的な用語も入れて」ではなく、層ごとの割合を表で渡します。

```
| 層 | 割合 | 重み |
|---|---|---|
| 基礎・一般（誰もが知る概念） | 25% | 7〜10 |
| 専門・中級（学んだ人が知る） | 40% | 4〜6  |
| ニッチ・辺縁（研究最前線）   | 35% | 1〜3  |
```

> **なぜ重要か:** 指定しないと重み7以上ばかりになり、バブルの多様性が失われます。

---

### 3. 強調したい領域を具体的な用語リストで示す

「詳しく書いて」ではなく、含めてほしい具体的な用語例を列挙します。

```
AIと認知の辺縁（各カテゴリに散りばめる）:
- AIの内部表現・解釈可能性（SAE、機械的解釈可能性、特徴量分解）
- スペキュラティブデコーディング・KVキャッシュ等の推論最適化
```

> **効果:** AI が「こういう粒度・方向性か」と文脈を理解し、類似するニッチ概念を補完生成します。

---

### 4. 禁止事項を明示する

出力の安定性のために、やってはいけないことを列挙します。

```
- `window.TOPICS_DB = [` から始め、最後の `];` まで省略なく出力
- 「// ...省略」「// 以下同様」などの省略は一切不可
- すべてのエントリを実際に記述すること
- 重複するキーワードは不可
```

---

### 5. 継続出力の指示を末尾に書く

大量生成では途中で止まることがあります。

```
途中で切れた場合は「続きを出力してください」と言えば継続すること。
```

**モデル別の対処:**

| モデル | 対処 |
|---|---|
| ChatGPT (GPT-4o / o3) | 「続きを出力してください」で継続。最後に「合計エントリ数を数えてください」で検証 |
| Gemini 1.5 / 2.0 Pro | 不安定な場合は「500エントリずつ分割して出力してください」を追加 |
| Claude | 「上記プロンプトで TOPICS.js を生成してください」でそのまま実行可 |

### 総エントリ数
**3000個前後**（2900〜3100の範囲）

### エントリの"深さ"の分布（重要）
単なる一般的・教科書的な用語が多すぎないよう、以下の比率を守ること：

| 層 | 割合 | 内容例 |
|---|---|---|
| 基礎・一般層（重み7〜10） | 25% | 量子力学、微積分、DNA |
| 専門・中級層（重み4〜6） | 40% | ゲルファント変換、デノボ合成、ポストコロニアル批評 |
| ニッチ・辺縁層（重み1〜3） | 35% | ベルヌーイ・スキーム、アルケー、タルスキの真理論 |

### 「ニュース」カテゴリ（新設・100〜150エントリ）
- 2025年5月時点の直近トピック（国際・国内・テック・経済・文化）
- 人名・企業名・政策名・事件名・新語も積極的に含める
- 重み設定: 話題の大きさに応じて1〜9

## index.html への組み込み

`topics.js` と `TOPICS.js` を両方読み込む場合、
`window.TOPICS_DB` の変数名が重複しないよう注意。
別変数名（例: `window.TOPICS_DB2`）にするか、生成後にマージする。

---

## topics.js の拡張手順

1. `generate-topics-prompt.md` を AI に貼り付けて実行
2. 生成されたコードを新ファイル（例: `topics2.js`）として保存
3. `window.TOPICS_DB` の変数名が重複する場合は `window.TOPICS_DB2` にリネーム
4. `index.html` で両ファイルを読み込み、JavaScript 側でマージ

```js
// index.html 内での結合例
const allTopics = [...(window.TOPICS_DB || []), ...(window.TOPICS_DB2 || [])];
```
